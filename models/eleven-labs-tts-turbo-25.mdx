## Overview

- **Provider**: ElevenLabs
- Submit via `POST https://api.wizzx.ai/api/v1/task/submit/eleven-labs-tts-turbo-25`; poll with `task_id` at `POST https://api.wizzx.ai/api/v1/task/status`.
- Required: `text` (1-5000 characters).
- Optional: `voice`, `stability`, `similarity_boost`, `style`, `speed`, `timestamps`, `previous_text`, `next_text`, `language_code`.
- **50% cheaper** than TTS V2, optimized for low-latency scenarios.

## Submit job

Endpoint: `POST https://api.wizzx.ai/api/v1/task/submit/eleven-labs-tts-turbo-25`, `Content-Type: application/json`

### Required

| Field | Type | Notes |
|------|------|------|
| `text` | string | Text to convert to speech (1-5000 characters) |

### Optional

| Field | Type | Default | Notes |
|------|------|---------|------|
| `voice` | string | `Rachel` | Voice selection (see voice list below) |
| `stability` | number | 0.5 | Voice stability (0-1) |
| `similarity_boost` | number | 0.75 | Similarity enhancement (0-1) |
| `style` | number | 0 | Style exaggeration (0-1) |
| `speed` | number | 1.0 | Speech rate (0.7-1.2) |
| `timestamps` | boolean | false | Return word timestamps |
| `previous_text` | string | - | Context before text (for continuity) |
| `next_text` | string | - | Context after text (for continuity) |
| `language_code` | string | - | ISO 639-1 language code |

## Query status

- Endpoint: `POST https://api.wizzx.ai/api/v1/task/status`
- Body:

```json
{
  "task_id": "task_123456789"
}
```

### Status response

```json
{
  "code": 0,
  "data": {
    "task_id": "task_123456789",
    "status": "SUCCEEDED",
    "result_urls": [
      "https://cdn.wizzx.ai/audio/output.mp3"
    ],
    "error": ""
  }
}
```

Status: `PROCESSING` (generating), `SUCCEEDED` (use `result_urls`), `FAILED` (`error` contains details).

## FAQ

- **When should I use Turbo 2.5 vs V2?** Use Turbo 2.5 for real-time applications, chatbots, or when cost is a concern. Use V2 for premium content requiring highest quality.
- **Is the quality noticeably different?** Turbo 2.5 offers high quality suitable for most use cases. V2 provides marginally better quality for audiophile-level requirements.
- **Can I switch between models?** Yes, both models use identical parameters. Simply change the endpoint.
