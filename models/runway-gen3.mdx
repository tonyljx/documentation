## Overview

- Proxy to Runway Gen-3 via Kie.ai; supports `text2video` and `img2video`.
- Every request creates an async video job; response returns `task_id` for polling.

## Endpoints

- `POST https://api.wizzx.ai/v1/video/runway-gen3` — submit a generation job.
- `GET https://api.wizzx.ai/v1/video/runway-gen3/{task_id}` — poll job status/result.

> Add your own prefix if needed; always send `Authorization: Bearer <token>`.

## Request schema (POST)

Field | Description | Rules / Default
-| - | -
`prompt` | Text prompt | Required
`mode` | Generation mode | Required: `text2video` or `img2video`
`input_image_url` | Reference image URL | Required for `img2video`, valid URL
`duration` | Seconds | Optional, default 5; allowed 5 or 10
`resolution` | Resolution | Optional, default `720p`; `720p` or `1080p` (10s does not support 1080p)
`fps` | Frame rate | Optional, default 24; `24`/`30`/`60`
`aspect_ratio` | Aspect ratio | Optional, `16:9`/`9:16`/`1:1` (empty = auto)

### Submit example

```bash
curl -X POST https://api.wizzx.ai/v1/video/runway-gen3 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "a cinematic drone flythrough of a neon city",
    "mode": "text2video",
    "duration": 5,
    "resolution": "720p",
    "fps": 24,
    "aspect_ratio": "16:9"
  }'
```

## Response format

On success:

```json
{
  "task_id": "rw_123",
  "status": "queued",
  "credits": 5,
  "duration": 5,
  "resolution": "720p",
  "mode": "text2video"
}
```

Status values: `queued` | `processing` | `succeeded` | `failed`.

## Polling for results

Use `GET /v1/video/runway-gen3/{task_id}` (poll every 2-5s):

```bash
curl -H "Authorization: Bearer $TOKEN" \
  https://api.wizzx.ai/v1/video/runway-gen3/rw_123
```

Success example:

```json
{
  "task_id": "rw_123",
  "status": "succeeded",
  "result": {
    "video_url": "https://cdn.example.com/rw_123.mp4",
    "cover_image_url": "https://cdn.example.com/rw_123.jpg"
  },
  "duration": 5,
  "resolution": "720p",
  "mode": "text2video"
}
```

On failure: `status: failed` with `error`.

## Pricing (reference)

- 5s 720p: 5 credits
- 5s 1080p: 10 credits
- 10s 720p: 5 credits
- 10s 1080p: not supported

> Charges depend on submitted duration and resolution; invalid combos fail validation.
