## Quick start

- Platform: Wizzx gateway; use `Authorization: Bearer <your_api_key>`.
- Submit: `POST https://api.wizzx.ai/api/v1/task/submit/runway-gen3`, async `task_id`.
- Poll: `POST https://api.wizzx.ai/api/v1/task/status` with `task_id`.
- Modes: `text-to-video` (text prompt only), `image-to-video` (1 reference image).

### Auth

```bash
Authorization: Bearer <your_api_key>
```

### Submit job

Endpoint: `POST https://api.wizzx.ai/api/v1/task/submit/runway-gen3`, `Content-Type: application/json`

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `prompt` | string | Yes | - | Text prompt describing the video |
| `type` | string | Yes | - | `text-to-video` or `image-to-video` |
| `image_urls` | array | Conditional | - | Image URLs (required for `image-to-video`, max 1) |
| `duration` | int | No | 5 | Video duration: `5` or `10` seconds |
| `resolution` | string | No | `720p` | `720p` or `1080p` (10s does not support 1080p) |
| `aspect_ratio` | string | No | auto | `16:9`, `9:16`, or `1:1` |

<Warning>
  **Constraint:** 10-second videos cannot use 1080p resolution.
</Warning>

#### Example: Text-to-Video

```bash
curl -X POST https://api.wizzx.ai/api/v1/task/submit/runway-gen3 \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "A cinematic drone flythrough of a neon city at night",
    "type": "text-to-video",
    "duration": 5,
    "resolution": "720p",
    "aspect_ratio": "16:9"
  }'
```

#### Example: Image-to-Video

```bash
curl -X POST https://api.wizzx.ai/api/v1/task/submit/runway-gen3 \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Make the character walk forward gracefully",
    "type": "image-to-video",
    "image_urls": ["https://example.com/character.jpg"],
    "duration": 5,
    "resolution": "720p"
  }'
```

Success response:

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "task_id": "task_123456789",
    "status": "pending",
    "credits_deducted": 5
  }
}
```

### Query status

Endpoint: `POST https://api.wizzx.ai/api/v1/task/status`

```json
{
  "task_id": "task_123456789"
}
```

Response (succeeded):

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "task_id": "task_123456789",
    "status": "succeeded",
    "result_urls": ["https://cdn.example.com/video.mp4"]
  }
}
```

Status flow: `pending` → `processing` → `succeeded` (returns `result_urls`) or `failed` (see `error`).

### Pricing

| Duration | Resolution | Credits |
|----------|------------|---------|
| 5s | 720p | 5 |
| 5s | 1080p | 10 |
| 10s | 720p | 5 |
| 10s | 1080p | Not supported |

### Best practices

1. **Duration & Resolution:**
   - For longer videos (10s), use 720p resolution
   - 1080p is only available for 5-second videos

2. **Mode Selection:**
   - Use `text-to-video` for pure text-driven generation
   - Use `image-to-video` when you have a reference image for better consistency

3. **Aspect Ratio:**
   - Use `16:9` for landscape videos
   - Use `9:16` for portrait/vertical videos
   - Use `1:1` for square videos
   - Leave empty for auto-detection (recommended for image-to-video)
