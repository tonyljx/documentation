## Overview

- **Breaking change (2025-12-10)**: `mode` → `quality_mode`; `image` → `image_urls` (array, 1 image); new `type` field (must be `image-to-video`).
- Submit via `POST https://api.wizzx.ai/api/v1/task/submit/kling-ai`; poll with `task_id` at `POST https://api.wizzx.ai/api/v1/task/status`.
- Required: `prompt`, `type="image-to-video"`, `image_urls` (1), `quality_mode` (`std`/`pro`), `duration` (5/10s).
- Optional controls: model version, negative prompt, cfg scale, tail frame, static/dynamic masks, camera control, callback, external task id.
- Mutual exclusion: only one of `image_tail`, `dynamic_masks/static_mask`, `camera_control` can be used.

## Auth

- Header: `Authorization: Bearer <your_api_key>`

## Submit job

Endpoint: `POST https://api.wizzx.ai/api/v1/task/submit/kling-ai`, `Content-Type: application/json`

### Required

| Field | Type | Notes |
|------|------|------|
| `prompt` | string | Positive prompt, ≤2500 chars |
| `type` | string | Generation type, **must be** `image-to-video` |
| `image_urls` | []string | Reference image URLs, **exactly 1** image |
| `quality_mode` | string | Quality mode: `std` (standard) or `pro` (professional) |
| `duration` | int | Video length: `5` or `10` seconds |

### Optional

| Field | Type | Default | Notes |
|------|------|--------|------|
| `model_name` | string | `kling-v1` | Versions: `kling-v1`, `kling-v1-5`, `kling-v1-6`, `kling-v2-master`, `kling-v2-1`, `kling-v2-1-master`, `kling-v2-5-turbo` |
| `image_tail` | string | - | Tail-frame control image URL; exclusive with brushes/camera |
| `negative_prompt` | string | - | ≤2500 chars |
| `cfg_scale` | float | 0.5 | Range [0, 1]; higher = closer to prompt |
| `static_mask` | string | - | Static brush mask URL |
| `dynamic_masks` | array | - | Up to 6 groups; each has `mask` + `trajectories` |
| `camera_control` | object | - | Camera motion; see details below |
| `callback_url` | string | - | Optional webhook |
| `external_task_id` | string | - | Custom task ID |

<Warning>
  Assets: jpg/jpeg/png, ≤10MB, width/height ≥300px, aspect ratio 1:2.5 ~ 2.5:1. `image_tail`, `dynamic_masks/static_mask`, and `camera_control` are mutually exclusive feature sets.
</Warning>

### Model versions

- `kling-v1`, `kling-v1-5`, `kling-v1-6`: base/iterative versions
- `kling-v2-master`, `kling-v2-1`, `kling-v2-1-master`, `kling-v2-5-turbo`: high-quality versions (recommended)

### Example: basic image-to-video

```bash
curl -X POST https://api.wizzx.ai/api/v1/task/submit/kling-ai \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "An astronaut stands up and walks away",
    "type": "image-to-video",
    "image_urls": ["https://example.com/astronaut.jpg"],
    "quality_mode": "std",
    "duration": 5,
    "cfg_scale": 0.5
  }'
```

## Query status

- Endpoint: `POST https://api.wizzx.ai/api/v1/task/status`
- Body:

```json
{
  "task_id": "local-task-id"
}
```

### Status response

```json
{
  "code": 0,
  "data": {
    "task_id": "local-task-id",
    "status": "SUCCEEDED",
    "result_urls": [
      "https://cdn.example.com/video.mp4"
    ],
    "error": ""
  }
}
```

Status: `PROCESSING` (submitted/processing), `SUCCEEDED` (use `result_urls`), `FAILED` (`error` contains details).

## Camera control

| type | Description |
|------|------|
| `simple` | Use `config` with one non-zero: `horizontal`/`vertical`/`pan`/`tilt`/`roll`/`zoom` (range -10 to 10) |
| `down_back` | Camera tilts down and moves back |
| `forward_up` | Camera moves forward and tilts up |
| `right_turn_forward` | Rotate right then move forward |
| `left_turn_forward` | Rotate left then move forward |

<Warning>
  `simple` mode: only one of the six `config` parameters can be non-zero.
</Warning>

## FAQ

- Why `mode` renamed to `quality_mode`? To avoid conflict with new `type`. `type` is the generation category; `quality_mode` is quality level (`std`/`pro`).
- Does Kling support text-to-video? No, `type` must be `image-to-video`.
- How to use camera control? Set `camera_control` with a supported `type`; `simple` uses `config` with exactly one non-zero field.
- Dynamic vs static masks? `static_mask` sets a fixed region; `dynamic_masks` add trajectory points for motion control.
- How long are videos retained? 30 days; please archive promptly.
