## Overview

- Submit via `POST https://api.wizzx.ai/api/v1/task/submit/kling-ai`, get `task_id` for polling.
- Modes `std` / `pro`, duration 5s or 10s; provide `image` or `image_tail` (at least one).
- Supports camera control, static/dynamic masks, tail-frame control; default `cfg_scale` 0.5.
- Status via `POST https://api.wizzx.ai/api/v1/task/status`.

## Auth

- Header: `Authorization: Bearer <your_api_key>`

## Submit job

Endpoint: `POST https://api.wizzx.ai/api/v1/task/submit/kling-ai`, `Content-Type: application/json`

### Required

- `prompt` (string): positive prompt, ≤2500 chars.
- `mode` (string): `std` or `pro`.
- `duration` (int): `5` or `10` seconds.
- `image` (string): reference image URL/Base64; `image` or `image_tail` required.

### Optional

- `model_name` (string): `kling-v1` / `kling-v1-5` / `kling-v1-6` / `kling-v2-master` / `kling-v2-1` / `kling-v2-1-master` (default `kling-v1`).
- `image_tail` (string): tail frame (URL/Base64). Mutually exclusive with `dynamic_masks` / `static_mask` / `camera_control`.
- `negative_prompt` (string): ≤2500 chars.
- `cfg_scale` (float): [0,1], default `0.5` (higher = closer to prompt).
- `static_mask` (string): static mask (URL/Base64), exclusive with `image_tail`/`camera_control`, must match aspect ratio.
- `dynamic_masks` (array): up to 6 masks; each has `mask` and `trajectories` (≤77 points for 5s; origin bottom-left).
- `camera_control` (object): camera motion. `type` = `simple` / `down_back` / `forward_up` / `right_turn_forward` / `left_turn_forward`. For `simple`, `config` has 6 params and only one can be non-zero (`horizontal`/`vertical`/`pan`/`tilt`/`roll`/`zoom`, range -10~10).
- `callback_url` (string): optional webhook.
- `external_task_id` (string): optional custom ID.

<Warning>
  Images: jpg/jpeg/png, ≤10MB, width/height ≥300px, aspect ratio 1:2.5 ~ 2.5:1. Provide at least one of `image` or `image_tail`.
</Warning>

### Example: basic image-to-video

```bash
curl -X POST https://api.wizzx.ai/api/v1/task/submit/kling-ai \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "An astronaut stands up and walks away",
    "mode": "std",
    "duration": 5,
    "image": "https://example.com/astronaut.jpg",
    "cfg_scale": 0.5
  }'
```

### Example: pro mode + camera control

## Query status

- Endpoint: `POST https://api.wizzx.ai/api/v1/task/status`
- Body:

```json
{
  "task_id": "local-task-id"
}
```

### Status response

```json
{
  "code": 0,
  "data": {
    "task_id": "local-task-id",
    "status": "SUCCEEDED",
    "result_urls": [
      "https://cdn.example.com/video.mp4"
    ],
    "error": ""
  }
}
```

Status: `PROCESSING`, `SUCCEEDED` (use `result_urls`), `FAILED` (see `error`).

## FAQ

- Callback required? No, poll `task/status`.
- Camera control? Use `camera_control`; for `simple`, only 1 non-zero config param.
- Dynamic vs static masks? `static_mask` is fixed; `dynamic_masks` include trajectories.
- `image_tail`? Controls tail frame; combine with `image` for first/last control.
- Model choice? `kling-v1` baseline; `v1-5/1-6` improved; `v2-*` latest/high quality.
- Retention? Videos stored 30 days; please archive promptly.
